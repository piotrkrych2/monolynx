## Project Conventions You Must Follow

### Database & Models
- UUID primary keys everywhere (never auto-increment)
- All DB columns for error data use PostgreSQL JSONB
- Soft delete pattern: `is_active = False`, filtered in all queries
- Use async SQLAlchemy sessions via `get_db()` dependency
- After `db.rollback()`, always re-query objects before passing to Jinja2 templates

### API & Routes
- Dashboard routes go in `src/monolynx/dashboard/<module>.py` with `router = APIRouter(prefix="/dashboard")`
- Register new routers in `dashboard/__init__.py` — static routes before dynamic `{slug}` routes
- Use `_get_user_id()` from `dashboard/helpers.py` for auth
- Slug validation: `^[a-z0-9]+(?:-[a-z0-9]+)*$`
- Pagination: query param `page` (int, default=1), fixed `per_page`; use `func.count()` then LIMIT/OFFSET
- Flash messages via `flash(request, message, type)`
- API key auth via `X-Monolynx-Key` header
- MCP auth via Bearer token (`osk_` prefix, SHA256 hashed in DB)

### Service Layer
- Services in `src/monolynx/services/` handle business logic
- Follow graceful degradation pattern (like embeddings.py, graph.py): `is_enabled()`, try/except, None fallback
- Use `ThreadPoolExecutor` for blocking I/O in async context
- Services must never crash the application

### Migrations
- Generate with `make migration msg="description"`
- **Always review autogenerated migrations** — they may include tables from previous migrations if DB was empty
- Each migration must only contain its own new/changed tables
- Alembic configured for async via `asyncio.run()` in `env.py`

### Code Quality
- Run `make lint` (ruff check --fix + ruff format + mypy) after changes
- Run `make test` or specific test files to verify
- All commands run inside Docker: `docker compose exec app <command>`
- Never run Python commands locally

### Constants & Config
- Shared constants in `src/monolynx/constants.py`
- Config via pydantic-settings in `src/monolynx/config.py`, reads env vars / `.env`
- UI text is in Polish

### Testing
- Tests in `tests/unit/` and `tests/integration/`
- Use `@pytest.mark.unit` or `@pytest.mark.integration` markers
- `login_session(client, db_session, email=...)` helper for auth in tests — use unique emails
- pytest-asyncio in auto mode with session-scoped loop
- `httpx.AsyncClient` with `ASGITransport` for API tests
