How to MCP

Podłącz Claude Code do Monolynx przez MCP i zarządzaj Scrumem z poziomu terminala

1

Wygeneruj token API

Przejdź do strony Tokeny API, nadaj tokenowi nazwę (np. "Claude Code") i kliknij Wygeneruj token.

Skopiuj token od razu — nie będzie wyświetlony ponownie.

2

Dodaj serwer MCP do Claude Code

Dodaj do pliku .mcp.json w katalogu projektu:

{ "mcpServers": { "monolynx": { "command": "npx", "args": [ "-y", "mcp-remote", "{{ app_url }}/mcp/", "--header", "Authorization: Bearer TWOJ_TOKEN" ] } } }

Zastąp TWOJ_TOKEN wygenerowanym tokenem API (zaczyna się od osk_).

3

Użyj narzędzi

Po połączeniu Claude Code ma dostęp do 35 narzędzi we wszystkich modułach:

Projekty

  • list_projects — lista projektów
  • get_project_summary — podsumowanie projektu

500ki (błędy)

  • list_issues — lista błędów
  • get_issue — szczegóły błędu
  • update_issue_status — zmień status
  • create_ticket_from_issue — utwórz ticket z błędu

Scrum — Tickety

  • list_tickets — lista ticketów
  • get_ticket — szczegóły ticketu
  • create_ticket — utwórz ticket
  • update_ticket — edytuj ticket
  • delete_ticket — usuń ticket

Scrum — Sprinty i tablica

  • list_sprints — lista sprintów
  • get_sprint — szczegóły sprintu
  • create_sprint — utwórz sprint
  • start_sprint — rozpocznij sprint
  • complete_sprint — zakończ sprint
  • get_board — tablica Kanban

Komentarze i czas pracy

  • list_comments — komentarze do ticketu
  • add_comment — dodaj komentarz
  • log_time — zaloguj czas pracy

Monitoring

  • list_monitors — lista monitorów
  • get_monitor — szczegóły monitora

Wiki

  • list_wiki_pages — drzewo stron
  • get_wiki_page — szczegóły strony
  • create_wiki_page — utwórz stronę
  • update_wiki_page — edytuj stronę
  • delete_wiki_page — usuń stronę
  • search_wiki — wyszukiwanie semantyczne (RAG)

Połączenia (graf)

  • create_graph_node — utwórz node
  • list_graph_nodes — lista node'ów
  • get_graph_node — szczegóły node'a
  • delete_graph_node — usuń node
  • create_graph_edge — utwórz krawędź
  • delete_graph_edge — usuń krawędź
  • bulk_create_graph_nodes — masowe tworzenie node'ów
  • bulk_create_graph_edges — masowe tworzenie krawędzi
  • query_graph — pobierz graf/podgraf
  • find_graph_path — najkrótsza ścieżka
  • get_graph_stats — statystyki grafu

Przykład użycia w Claude Code:

"Stwórz ticket 'Naprawić bug logowania' w projekcie my-app z priorytetem high" "Pokaż wszystkie tickety w sprincie" "Dodaj komentarz do ticketu #3: poprawka wdrożona na produkcję"
4

Zainstaluj skille Claude Code

opcjonalnie

Skille to gotowe przepisy pracy dla Claude Code. Pobierz paczkę i rozpakuj w katalogu głównym projektu — pliki trafią do .claude/skills/.

Pobierz monolynx-skills.zip

Po rozpakowaniu zamień <SLUG_PROJEKTU> na slug Twojego projektu w Monolynx (np. my-app).

monolynx-work

Koordynator zespołu agentów — podejmuje ticket ze sprintu, rozdziela pracę, loguje czas.

Użycie: /monolynx-work [ticket-id]

monolynx-search

Wyszukiwanie semantyczne w wiki projektu (RAG). Aktywuje się automatycznie na pytania o dokumentację.

Użycie: /monolynx-search jak działa autoryzacja w API?

monolynx-create-graph-ci-script

Generuje skrypt CI synchronizujący graf zależności kodu z Twojego projektu.

Użycie: /monolynx-create-graph-ci-script
